# Talk to Claw — Voice Calling for OpenClaw

## Installing this skill

```bash
# Download
curl -L "https://github.com/sinkers/voice-agent/raw/master/web-skill/talk-to-claw.skill" \
  -o talk-to-claw.skill

# Install (per-agent workspace)
unzip talk-to-claw.skill -d ~/.openclaw/workspace-<agent-id>/skills/

# Install (shared across all agents)
unzip talk-to-claw.skill -d ~/.openclaw/skills/
```

Then start a new session — the skill will be available immediately.

---

## What This Does

Deploys a Fly.io-hosted web app so anyone can talk to an OpenClaw agent by clicking a link. The agent on this machine handles calls — no manual terminal commands needed.

- **React frontend** — connect-to-call UI with agent state (Listening / Thinking / Speaking)
- **FastAPI backend** — verifies signed call URLs, issues LiveKit tokens, dispatches the agent
- **Agent worker** — runs on this OpenClaw machine, connects to LiveKit Cloud to handle voice

## Before You Start

### Required accounts

| Service | What it's for | Sign up | Free tier? |
|---------|--------------|---------|-----------|
| **LiveKit Cloud** | WebRTC infrastructure | https://cloud.livekit.io | ✅ Yes |
| **Deepgram** | Speech-to-text | https://console.deepgram.com | ✅ Yes ($200 credit) |
| **OpenAI** | LLM + TTS (alloy voice) | https://platform.openai.com | ❌ Pay per use |
| **Fly.io** | Hosts the web app | https://fly.io/app/sign-up | ✅ Yes |

### Credentials

Setup will prompt for:
```
LIVEKIT_URL          wss://your-project.livekit.cloud
                     → LiveKit Cloud → Project Settings → URL

LIVEKIT_API_KEY      APIxxxxxxxxxxxxxxxxx
                     → LiveKit Cloud → Project Settings → API Keys

LIVEKIT_API_SECRET   (shown once at key creation)
                     → LiveKit Cloud → Project Settings → API Keys
```

These go in the agent `.env` file separately (setup won't ask for them):
```
DEEPGRAM_API_KEY     → console.deepgram.com → API Keys
OPENAI_API_KEY       → platform.openai.com/api-keys
```

`CONFIG_SECRET` is auto-generated — you don't need to create it.

## Setup

Run setup from the voice-agent repo directory:

```bash
python3 web-skill/scripts/setup.py
```

Or if running from the skill install directory:
```bash
python3 scripts/setup.py --repo ~/path/to/livekit-agent
# or: TALK_TO_CLAW_REPO=~/path/to/livekit-agent python3 scripts/setup.py
```

Setup will:
1. Check for flyctl and Node.js (installs flyctl automatically if missing)
2. Authenticate with Fly.io
3. Prompt for LiveKit credentials
4. Deploy the web app to Fly.io
5. **Start the agent worker automatically**
6. **Print your call URL** — ready to share immediately

### Sharing one Fly app across multiple OpenClaw agents

If the Fly app already exists (another agent deployed it), setup will prompt for the existing `CONFIG_SECRET` so both agents can generate valid URLs against the same backend. Enter the existing secret — do not generate a new one.

## Generating Call URLs

The agent worker prints a 24h call URL at startup. To generate a new one at any time:

```bash
python3 web-skill/scripts/call_url.py --agent voice-agent --name "Alex"
```

Pass `--ttl 3600` for a 1-hour URL. The agent name must match the registered worker name (shown in the startup log).

## Environment Variables

These live in the agent's `.env` file in the repo root:

| Variable | Description |
|----------|-------------|
| `LIVEKIT_URL` | LiveKit WebSocket URL (`wss://...`) |
| `LIVEKIT_API_KEY` | LiveKit API key |
| `LIVEKIT_API_SECRET` | LiveKit API secret |
| `OPENAI_API_KEY` | OpenAI key (TTS always uses this directly) |
| `DEEPGRAM_API_KEY` | Deepgram key for speech-to-text |
| `CONFIG_SECRET` | Shared JWT signing secret (auto-generated by setup) |
| `CALL_BASE_URL` | Web app URL (set by setup) |
| `OPENCLAW_AGENT_NAME` | Base agent name (default: `voice-agent`) |
| `OPENCLAW_AGENT_DISPLAY_NAME` | Display name shown in the UI |
| `OPENCLAW_AGENT_ID` | OpenClaw agent ID (routes LLM through Gateway) |
| `OPENCLAW_SESSION_KEY` | Pins voice calls to a specific OpenClaw session |
| `AGENT_HTTP_PORT` | LiveKit worker HTTP port (default: `8081`) |
| `AGENT_GREETING` | Text spoken on connect via TTS |

## Troubleshooting

**"Invalid config token" / 401** — `CONFIG_SECRET` mismatch between `.env` and Fly. Re-run setup with `--update-secrets`:
```bash
python3 scripts/setup.py --update-secrets
```

**"Waiting for agent to join" never resolves** — the agent worker isn't running or isn't registered with LiveKit. Check the agent log for `registered worker`. The agent worker is managed by the OpenClaw agent on this machine — ask it to check status or restart.

**Call URL says "expired"** — generate a fresh URL. The agent worker prints a new URL each time it starts; you can also generate one on demand with `call_url.py`.

**Agent responds but ignores context** — check `OPENCLAW_SESSION_KEY` in `.env` is set to the correct session key for this agent.
